# AIBOUX Short URL And Bark Final-Only Rule

作成日: 2026-05-31

## Status

PREVIEW_READY

## Rules

- User-facing reports must use short URLs only.
- Long preview URLs, query URLs, and tunnel URLs with long paths are internal verification details only.
- URLs must appear only in the final `URLs` section.
- No text may follow the `URLs` section.
- Bark notifications are final-only and must not be sent for CODE_READY, PREVIEW_READY, DEPLOYED, BLOCKED, or USER_ACTION_REQUIRED.

## Short URL Roles

- `mNN`: master update preview
- `lNN`: execution log preview
- `dNN`: screen or artifact preview

The same report must use the same number for all three URLs.

## Current Bundle

- `m70.html`: master update preview
- `l70.html`: execution log preview
- `d70.html`: delivery detail screen preview

## Required Verification

- `m70.html` HTTP 200
- `l70.html` HTTP 200
- `d70.html` HTTP 200
- `d70.html` directly contains the delivery detail screen
- final report is generated by `scripts/render-aiboux-report.mjs`
- final report contains short URLs only
- final report puts URLs last
- `npm run check:control-chars` PASS
- `npm run check:mojibake` PASS
- `npm run check:bark-policy` PASS

## Bark

- notification: not sent
- reason: Bark notifications are final-only by policy.
- secretLogged: false
